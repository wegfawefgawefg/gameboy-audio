How to code GameBoy music.

Noise is a stream of rough high/low offsets.

Alright so if you stick to two sequences of notes stacked on top of eachother, each using one voice, where one is a square wave and the other can be square or pulse, youre in real Game Boy territory. But what about drums and stuff. Well thats where noise comes in.

Noise is just messy speaker offsets. Instead of drawing a clean repeating square wave, we keep throwing random-looking numbers into the sample list.

Random can be a voice too lol.

import random

sample_rate = 44100
seconds = 0.7
volume = 0.18

num_samples = int(seconds * sample_rate)
samples = []

for sample_index in range(num_samples):
    random_offset = random.uniform(-1, 1)
    sample = random_offset * volume
    samples.append(sample)

See thats already pretty good for noise, but it doesnt really give us a lot of character knobs. We can make it quieter or louder, and we can fade it in or out, but thats about it.

The Game Boy does not ask Python for a true random number. It has a tiny bit machine. That sounds worse, but its useful: the hardware can run it cheap, and the noise has knobs. You can clock it faster or slower, so the noise gets brighter, duller, tighter, or chunkier.

The bit machine is called an LFSR. That stands for linear feedback shift register. Gross name. It just means we keep one integer around, pull some bits out of it, combine them, shift the integer over, and feed the new bit back in. That gives us a long stream of high/low offsets that feels noisy.

The useful knobs are the clock speed, the feedback tap, and the volume shape. Clock speed changes how often the noise value updates. Feedback tap changes the pattern of bits. Volume shape decides whether it is a constant hiss or a little hit that dies away. 

sample_rate = 44100
seconds = 0.8
volume = 0.30
num_samples = int(seconds * sample_rate)

samples = []

timer = 0
current_offset = 1
noise_clock_hz = 5000
feedback_tap = 1
lfsr = 0b111_1111_1111_1111

for sample_index in range(num_samples):
    # Audio samples still happen 44,100 times per second.
    timer = timer + 1 / sample_rate

    # But the noise bit only changes when this clock ticks.
    # Higher clock = brighter fizz.
    # Lower clock = rougher chunk.
    if timer >= 1 / noise_clock_hz:
        timer = timer - 1 / noise_clock_hz

        # Take two bits from the register.
        # XOR them together.
        # That result becomes the new incoming bit.
        bit = (lfsr ^ (lfsr >> feedback_tap)) & 1

        # Shift everything one slot to the right,
        # then put the new bit back on the left.
        lfsr = (lfsr >> 1) | (bit << 14)

        # The bottom bit becomes our speaker offset.
        current_offset = -1 if lfsr & 1 else 1

    # Constant noise is just hiss.
    # Fade it down and now it starts acting like a tiny drum hit.
    time = sample_index / sample_rate
    fade = max(0, 1 - time / 0.45)

    sample = current_offset * volume * fade
    samples.append(sample)

Heres the bit machine actually doing the thing. The line picks two bit slots, XORs them into a new bit, the register shifts right, and the bottom bit becomes the next high/low sample.

GAY BOY

Thats basically the point of the Game Boy noise channel. Channel 4 does not play normal melody notes like the pulse channels. It clocks this little LFSR thing and turns the bit stream into high/low offsets. The clock settings are the character knobs. Not magic. Just tunable trash.

The real Game Boy exposes those knobs through its noise registers. You do not send it a huge list of random samples. You set parameters like clock rate, width mode, and envelope, and the chip spits out the noisy high/low stream for you.

But in software we are not stuck with the Game Boy circuit. XOR is special because it tends to make long useful bit patterns. But you can put other little logic functions in the feedback path and listen to the machine get weird, collapse, buzz, or turn into a different texture. 

def feedback(mode, a, b, c):
    if mode == "xor":
        return a ^ b
    if mode == "nand":
        return 1 - (a & b)
    if mode == "xor_or":
        return (a ^ b) | c
    if mode == "majority":
        return 1 if a + b + c >= 2 else 0
    if mode == "parity3":
        return a ^ b ^ c


new_bit = feedback(mode, bit_0, bit_tap_a, bit_tap_b)

Then theres the other chain of thought. What if the register is not bits. What if it is a row of smooth numbers, and the feedback function is something soft like tanh, sin, or a folded curve. Now it is not a Game Boy noise channel anymore. Its a little procedural noise synth. 

def soft_xor(a, b, c, drive):
    return tanh((a - b) * drive)


def sine_mix(a, b, c, drive):
    return sin(a * drive + b * drive * 0.7 + c * drive * 0.35)
