Game Boy audio, built from first principles

We are going to explain how the Game Boy makes sound, then write code that does the same kind of thing.

We will ignore transistor-level electronics unless they help the mental model. The first model is simple: a sample is one desired speaker offset from center.

1. Samples Move The Speaker

The graph is a list of samples. Playback scans across the dots. The speaker uses the same current dot as its offset from center.

The Code Idea

We are building a waveform: a list of offsets known as samples. Each sample is one desired speaker offset at one specific time.

Spelled Out

from math import pi, sin

sample_rate = 44100
# The number of speaker offsets we make in one second.
#
# 44,100 is common because CD audio used it.
# More samples per second gives higher time resolution.
# Fewer samples per second gives lower time resolution.
#
# If the rate is too low, high frequencies cannot be represented.
# If the rate is too high, we waste memory and processing.

frequency = 220
# How many back-and-forth cycles happen per second.
#
# frequency = 1 means one cycle per second.
# frequency = 10 means ten cycles per second.
# frequency = 220 means 220 cycles per second.

volume = 0.4
# Magnitude scaler.
#
# volume = 0.4 makes every offset 40% as large.
# volume = 1.0 leaves offsets unchanged.
# volume = 2.0 makes every offset twice as large.

seconds = 1
# How many seconds worth of samples do we need?

num_samples = seconds * sample_rate
# How many total offsets do we need?
#
# 1 second worth of samples:
# 1 * 44,100 = 44,100 samples.
#
# 10 seconds worth of samples:
# 10 * 44,100 = 441,000 samples.
#
# 60 seconds worth of samples:
# 60 * 44,100 = 2,646,000 samples.

samples = []
# samples will become our waveform:
# a list of speaker offsets.

for sample_index in range(num_samples):
    # Which moment in time is this sample for?
    #
    # sample_index = 0 is time 0.0 seconds.
    # sample_index = 22050 is time 0.5 seconds,
    # halfway through the first second.
    # sample_index = 44100 is time 1.0 seconds.
    time = sample_index / sample_rate

    # Make one raw offset for that moment.
    #
    # frequency * time tells us how many cycles have passed.
    #
    # If frequency = 220 and time = 0.5,
    # then 110 cycles have passed.
    #
    # sin() wants an angle, not a cycle count.
    # One full cycle around a circle is 2 * pi radians.
    # So 2 * pi * frequency * time converts cycles into an angle.
    sample = sin(2 * pi * frequency * time)

    # Scale the offset magnitude.
    # With volume = 0.4:
    # +1.0 becomes +0.4. -1.0 becomes -0.4.
    sample = sample * volume

    # Store this offset in the waveform list.
    samples.append(sample)

Flat Code

from math import pi, sin

sample_rate = 44100
frequency = 220
volume = 0.4
seconds = 1
num_samples = seconds * sample_rate
samples = []

for sample_index in range(num_samples):
    time = sample_index / sample_rate
    sample = sin(2 * pi * frequency * time)
    sample = sample * volume
    samples.append(sample)

Multiplying by volume scales the offset. A volume below 1.0 makes the movement smaller. A volume above 1.0 makes it larger. Same shape, different speaker movement size.
