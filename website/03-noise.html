Chapter 3

Noise is a fast stream of not-quite-random offsets.

The Game Boy noise channel uses an LFSR. For a coder, think of it as a tiny bit machine that keeps producing high/low values.

Why Noise Matters

Short noise bursts become percussion: hi-hats, snares, clicks, explosions, wind, and texture. You do not need a drum sample. You need a noisy offset pattern and a volume shape.

Spelled Out

sample_rate = 44100
seconds = 1
volume = 0.35
num_samples = seconds * sample_rate

samples = []

# LFSR means linear feedback shift register.
# It is a small integer where bits are shifted each step.
lfsr = 0b111_1111_1111_1111

# The noise does not change every audio sample.
# It changes when this smaller clock ticks.
noise_clock_hz = 5000
timer = 0
current_offset = 1

for sample_index in range(num_samples):
    timer = timer + 1 / sample_rate

    if timer >= 1 / noise_clock_hz:
        timer = timer - 1 / noise_clock_hz

        # Take two bits, xor them, and feed the result back in.
        bit = (lfsr ^ (lfsr >> 1)) & 1
        lfsr = (lfsr >> 1) | (bit << 14)

        if lfsr & 1:
            current_offset = -1
        else:
            current_offset = 1

    # Fade out so it sounds like a tiny drum hit.
    time = sample_index / sample_rate
    fade = max(0, 1 - time / 0.45)
    samples.append(current_offset * volume * fade)

Game Boy Connection

Channel 4 is the noise channel. It does not play a normal pitch like the pulse channels. It clocks an LFSR and turns the bit stream into high/low speaker offsets. The clock settings change whether the noise sounds lower, higher, harder, or softer.
