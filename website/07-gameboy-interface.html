How to code GameBoy music.

Now bottle the ideas in a Game Boy-like interface.

The Four Channels

CH1 Pulse + Sweep

Pulse wave, duty, envelope, and pitch sweep.

CH2 Pulse

Pulse wave, duty, and envelope. Great for a second voice.

CH3 Wave

A tiny custom waveform table: 32 samples, 4 bits each.

CH4 Noise

LFSR noise for percussion and texture.

A Software Interface In The Same Spirit

pulse_1 = {
    "enabled": True,
    "frequency": 440,
    "duty": 0.5,
    "volume": 0.3,
    "sweep": None,
}

pulse_2 = {
    "enabled": True,
    "frequency": 660,
    "duty": 0.25,
    "volume": 0.2,
}

wave = {
    "enabled": True,
    "frequency": 110,
    "volume": 0.25,
    "table": [-1, -0.5, 0, 0.5, 1, 0.5, 0, -0.5],
}

noise = {
    "enabled": False,
    "clock_hz": 5000,
    "volume": 0.25,
}

This is not hardware-accurate yet. It is a teaching interface: each channel has the few settings we already explained.

Hardware Shape

CH1: pulse + sweep
CH2: pulse
CH3: wavetable
CH4: noise

To start a pulse note on real hardware:
  write duty and length
  write envelope
  write frequency low bits
  write frequency high bits with trigger = 1

That register interface comes later. The important thing now is that all four channels still reduce to offset patterns over time.
